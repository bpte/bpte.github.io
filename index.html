##堆结构
1.堆结构就是用数组实现的完全二叉树结构。
2.完全二叉树中如果每颗子树的最大值在顶部就是大根堆。
3.完全二叉树中如果每颗子树的最小值在顶部就是小根堆。
4.堆的heapinsert和heapify操作。
5.堆结构的增大和减少。
6.优先级队列结构，就是堆结构。

###大根堆

假设装有大根堆的数组下标为i,如果这个数组从arr[0]开始装载大根堆的数据，那么：
大根堆左孩子的下标= 2*i+1
大根堆右孩子的下标= 2*i+2
大根堆父节点的下标= (i-1)/2

如果数组从arr[1]开始装载大根堆的数据，那么：
大根堆左孩子的下标= 2*i
大根堆右孩子的下标= 2*i+1
大根堆父节点的下标= i/2
此时这些数据的下标也可以写成：
大根堆左孩子的下标= i<<1
大根堆右孩子的下标= i<<1|1
大根堆父节点的下标= i>>1
这样写的优点是将乘法除法运算都改成了位运算，计算机的运行速度更快

要求1：
依次给你数字，要求把这些数字组织成大根堆形式

要求2：
求出目前大根堆上最大值并返回，然后将最大值从大根堆上删去并使剩下的数据依旧保持大根堆

结构体数组

结构体数组是指在一个数组中，所有的元素都是同一结构体类型的变量。

在实际应用中，经常使用结构体数组来表示具有相同数据结构的一个群体。

结构体数组和整型数组都差不多，不同的地方就是在于每一个数组的元素都是一个结构体数组，他们都有各自的成员项。

定义：结构体数组的定义与结构体变量类似，只需要说明它为数组类型即可。

lielly/unlikely函数

likely/unlikely函数

likely()与unlikely()是内核中定义的两个宏。位于/include/linux/compiler.h中
具体定义如下：

我们可以看到其实likely与unlikely函数就是对_builtin_expect的一个封装而已，_builtin_expect这个指令是gcc引入的。该函数作用是允许程序员将最有可能执行的分支告诉编辑器，来辅助系统进行分支预测。

它的用法为： _builtin_expect(EXP,N)。意思是：EXP == N的概率很大。那么likely与unlikely这两句的具体含义是：
_builtin_expect(!!(x),1)x为真的可能性更大
_builtin_expect(!!(x),0)x为假的可能性更大

当正确的使用了 _builtin_expect之后，编译器会在编译过程中根据程序员的指令，将可能性更大的代码紧跟着前面的代码，从而减少指令跳转带来性能的下降

Note:
1.之所以使用likely，unlikely函数来提高CPU的运行速率是因为这两个函数可以改变汇编语言跳转指令后面的分支，使用likely函数可以让跳转指令后面的分支是x为真的分支；而unlikely正好相反，它可以让跳转指令后面的分支是x为假的分支。
2.CPU在执行程序指令的时候，并不是限制性一个，执行完了再执行下一个这样的，而是把每个指令分成了多个阶段，并让不同指令的各部操作重叠，从而实现几条指令并行处理。执行每条指令都有取址，译码，执行，写回四条指令。由之前所知，在跳转指令正在被CPU执行的情况下，在跳转指令下面两条指令的取址与译码都已执行好了。假如说分支预测失败，那么对跳转指令性下的取址与译码指令就白做了。
3.我们身为程序员，当然清楚跳转指令下面的指令哪个更容易被选择，我们就可以使用likely与unlikely函数来改变跳转指令下面的汇编语句，从而达到提高CPU运行的速度。

sprintf函数

sprintf函数

C语言头文件： <stdio.h>

用法：int sprintf(char *str, const char*format,... )

功能：将格式化的数据写入某个字符串缓冲区。

返回值：字符串长度

sprintf函数与printf函数的区别:
sprintf的作用是将一个格式化的字符串输出到一个目的字符串中，而printf的作用是将一个格式化的字符串输出到屏幕。sprintf的第一个参数应该是目的字符串，如果不指定这个参数，执行过程中出现”该程序产生非法操作，即将被关闭……”的提示。

sprintf常见应用：
sprintf(s, "%d", 123); //把整数123打印成一个字符串保存到s中

Note：
C语言对数组操作时不检查数组的长度，如果str的长度不够，sprintf()很容易造成缓冲溢出，带来意想不到的后果，黑客们常常用这个弱点去攻击看上去安全的系统。

strncpy函数

strncpy函数

C语言头文件： <string.h>

用法： char* strnpy(char *dest, const char* src，int n);

功能：把src指向的字符串复制到dest,最多复制n个字符，当src的的长度小于n时，dest将剩余部分用空字节填补。

Note:

1.当src字符串长度小于n时，在拷贝完字符串后，剩余部分用空字节填补，直到n个，strnpy不会向dest追加’\0’。
2.src和dest区域不能重叠，且dest必须有足够的空间放置n个字节。

strncpy函数与strcpy函数的区别

1.strcpy就是用来复制字符串的，而strncpy函数用来复制字符串的前n个字符。
2.不像strcpy函数，strncpy函数不会向dest追加’\0’。

strcpy函数

strcpy函数

C语言头文件: <string.h>

用法： char *strcpy(char *destination, const char *source);

功能：strcpy函数将含有’\0’结束符的字符串复制到另一个地址空间

返回值：返回的是目标字符串的首地址

Notes：
1.目标空间必须足够大，以确保能够放得下复制过来的字符串。
2.目标空间必须可变，即 char *destination不能用const修饰，并且目的地空间的字符串不能时常量字符串，常量字符串不能被修改。
3.源字符串必须含有’\0’。
4.strcpy函数会将源字符串的’\0’也拷贝到目的地空间

返回值：返回的时目的字符串的首地址。
