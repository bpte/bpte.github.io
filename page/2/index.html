假如我在程序中声明了一个变量x,然后初始化为5；程序运行的时候，会为这个变量分配一些内存，整型一般会被分配4个字节的地址，我们假设从225开始的四个字节分配给了x，里面存的值为5。为了存放整型变量x的地址，我们需要声明一个指向整型的指针p，然后把*放在p的前面。现在的情况是，编译器会为这个变量p预留一些内存空间，指针也占4个字节，假设把从215开始的四个字节分配给p，然后将x的地址存入p(也就是p指向了x),取地址符&得到变量的地址；这里重要的一点是，我们之所以能够把x的地址存入p是因为p是一个指向整型的指针，如果p指向字符型或其他的类型，那么(*p=&x;)这句就不合法了。

为什么我们需要指向整型的指针来保存整型数据的地址？
是因为我们不仅仅保存地址，我们还使用指针来对地址进行解引用和写数据，因此如果我写了这条语句(*p = 6),那么地址225里面，x被修改了。

那我是否可以创建一个指针指向变量p，p本身也是一个指向整型的指针？
是的，我们可以这样做。比如我们创建一个变量q,来存储p的地址，那么这个q是什么类型的呢？
我们需要一个特定类型的指针，来存放特定类型的地址，因此为了存放一个指针，我们需要一个指向指针的指针，所以我们在变量面前放两个**，现在q可以存放p的地址了。

我们创建了一个变量q，假设q的地址为205，于是q里面存放了p的地址，q指向了p,q的类型是**q。
简单的一种解读方式是：

  • 变量x是整型，为了得到x的地址，我们需要int*类型的指针，我们放了一个星号在p前面，代表p指向了q。
  • 为了得到p的地址，我们需要int**类型的指针，我们放了两个星号在q的前面，代表q指向了p。
  • 如果我们需要生成n级指针，我们就需要在变量前面加(n+1)个*。

代码运行原理解释：
当一个程序启动的时候，计算机会设置(或预留了一些内存)给这个程序使用,应用程序使用的内存通常被划分为4个部分：

1.Code(Text)： 这部分内存是用来存储程序的指令，计算机需要把指令加载到内存，像程序中的自增程序等等，他们都是串行的指令，都会是内存的一部分。
2.Static/Global： 这部分内存是用来存储静态或全局变量，如果我们不是在函数中声明变量，那么它将是一个全局变量。作为一个全局变量，在程序的任何地方，都可以访问和修改，不像局部变量，只能在特定的函数和代码块进行访问和修改。
3.Stack：这部分内存的名称是栈，局部变量都存放到这个地方。
4.Heak：这部分内存的名称叫做堆。

前三个内存空间是固定的，在程序开始运行的时候就已经确定了，但是应用程序在运行的时候可以要求堆区域为它分配更多的内存空间。

那么程序到底是怎样运行起来的呢？

我们都知道，内存中的每个字节都是可以寻址的，假设我们程序被分配的内存空间ROM是从200到800，在200到800的地址中就有许多的段，我们假设地址300到600是栈区。现在，当一个函数被调用的时候，像是程序开始运行的时候，main函数会被调用，所有的关于这个函数的信息，像是参数，所有的局部变量，返回的地址，当前执行的指令等，所有的这些信息都存在栈里。所以在栈里，会开辟出一片空间给main函数，创建出一块区域，我们称为stack frame，每个函数里都将会有一个栈帧。现在我们有一个变量’a’，给a分配内存，并赋值给10。现在main函数调用Increment这个函数的时候，其实事情是这样发生的：机器就好像在说，我会暂停你的执行，我会在这条指令的地方停下来，先去执行Increment函数，一旦结束之后，我就会再次回到main函数，现在会为Increment函数再开辟一个栈帧，Increment函数使用的参数，就像我们这里的变量’a’，都会相应的分配到新的内存空间，传进来的值会被拷贝到新的内存空间，当我们写下a = a + 1这条语句，实际上发生的事情是这个在Increment函数里的局部变量’a’,在这个特定的栈帧里，’a’增大了1，他不能访问自己栈以外的变量，当Increment函数调用完毕，程序控制流会再一次返回main函数，机器所做的就是清除之前那个Increment函数所使用的栈帧，然后main函数从之前暂停的那个地方恢复执行，局部变量的生命周期，在函数执行期间一直存在，main函数的下一个语句是printf，printf他不是用户定义的函数，它是一个库函数，main函数的执行被暂停，现在转而执行printf，我们通常称这种特殊的结构，为调用栈或者函数调用栈，栈顶的函数正在执行，但是栈顶是有固定大小的。所有，如果有这样一个场景，一个函数无限次的调用另一个函数，就像是无限递归，那么栈会溢出，系统会崩溃，但是这与我们现在写的代码没有太大关系。但是你必须要有一个概念，当一个函数调用另一个函数的时候会发生什么事情。Increment(a)这句里面的变量’a’是在main栈帧里面的，main是主调函数，而Increment是被调函数，当我们在主调函数中调用其他函数时，写进去的这个参数被叫做实参，也就是说Increment(a)这句里面的变量’a’就是实参，被调函数中定义的参数被叫做形参，也就是说void Increment(int a)代码中的a就是形参。实参会被映射到形参，当这个函数调用发生的时候，实参’a’会被映射到另一个形参’a’,如果void Increment(int a)里面的形参不是’a’而是’x’,在Increment函数里面的代码也改成x = x + 1,那么’a’就会被映射到’x’,也就是说变量’a’的值会被拷贝到变量’x’。当我们做一个函数调用的时候，我们基本上就是把一个变量映射到另一个变量，这种形式的函数调用被称为传值调用。

##goto语句
goto语句又叫无条件转移语句，它可以跳转到任何一行代码前面，这使得他的功能也显得格外强大，但是在这个强大的功能下伴随着巨大的风险，goto语句会造成逻辑上的混乱，使得代码会出现许多意想不到的错误，而且排查这些错误的时候也显得格外困难。因为它有可能会跳过某些结构体的构造、变量的初始化已经重要的计算语句，因此能不使用goto语句就尽量不使用goto语句。

但是goto语句也有自己的使用场景，但是也就局限于那一两种场景，像那种从一个函数跳转到另外一个函数的这种用法是绝对禁止的。

资源的动态申请和释放模板
对于一个程序来说如果有许多模块进行动态申请初始化，假如有的模块初始化不成功，那就必须将之前动态分配的空间给释放掉(如果不释放掉就会造成内存泄漏)，这时候一个一个释放就使得代码特别的臃肿，所以我们就可以使用goto语句跳转到一个固定的地方同意释放内存，这样使得代变得简洁。

实际上，goto语句是C语言为底层造轮子语言的一个象征，为什么这么说呢？

功能：assert函数用于判断传入的expression是否为真，如果为真，assert函数不起作用；如果为假，打印一条出错程序，然后调用abort函数来终止程序运行。实际上，如果NDEBUG这个宏没有被定义，assert才会被调用；如果NDEBUG这个宏被定义了，assert函数则不起作用。

为什么assert函数要在NDEBUG没有定义的情况下才会被调用呢？
实际上是为了用户体验才这样定义的，如果一出问题就直接终止程序，那么用户的体验肯定不好，我们需要的是即使出问题了程序也要运行，但是会有提示信息。这种情况下就不需要使用assert函数来终止程序，因此在release版本中NDEBUG就会被定义，从而使得编译器不会执行assert函数。

###随机数的本质
多次运行以上的代码，你会发现得到的值是一致的。实际上，rand()函数产生的随机数都
是伪随机数，是根据一个数值按照某一个公式推导出来的，这个数值被我们称为“种子”。

种子每次启动时都是随机的，但是计算机一旦启动后这个值就不会再变了化了，也就是说，
每次启动计算机后，种子的值都是固定的了，所以根据公式推出来的结果也是固定的了。

###播种
我们可以通过srand()函数进行重新“播种”，这样种子就会变化。srand()的用法为：
void srand(unsighed int seed)

它需要一个unsighed int类型的参数。通常情况下，我们使用时间作为参数，只要每次播种的时间不同，播种出来的结果(即随机数)自然也不相同。

使用<time.h>头文件里的time()可以得到目前的时间(精确到秒)，就像下面这样：
srand((unsighed)time(NULL))

此类递归的时间复杂度计算方法为：$T(N)= a T(\frac{N}{b}) + o(N^d)$
a：一个递归函数内部调用自己的次数，上述题目的a=2
b：将问题切分的规模,上述题目每次调用都将数组二分，所以b=2
d：除调用自己以外所有操作之和的时间复杂度，上述题目其他操作的时间复杂度为o(1),所以d=1;
所以上述题目的时间复杂度为：
$T(N)= 2 T(\frac{N}{2}) + o(N^0)$

如果递归符合$T(N)= a T(\frac{N}{b}) + o(N^d)$这个式子的话
1.如果 $\log_b a$ > d;则时间复杂度为o($N^{\log_b a}$)
2.如果 $\log_b a$ < d;则时间复杂度为o($N^{d}$)
3.如果 $\log_b a$ = d;则时间复杂度为o($N^{d}*logN$)

##一、衡量算法的标准：

  • 时间复杂度（流程决定）
  • 额外空间复杂度（流程决定）
  • 常数项时间（细节决定）

###1.时间复杂度：

####算法流程的时间复杂度的确定：
当完成了表达式的建立，只需要留下最高阶项即可，低阶项与最高阶项的系数也去掉。
我们把它记为o(忽略掉系数的最高项)。

####时间复杂度的意义：
当我们需要测试的数据格外庞大时，常数项系数与低阶项就变得无关紧要了，而最高项就显得尤为重要。
时间复杂度是衡量算法流程的复杂程度的一种指标，该指标只与数据量有关，与过程的优化无关。

####常见的时间复杂度：
以下时间复杂度从低到高

  1. o(1)
  2. o(log n)
  3. o(n)
  4. o(n * log n)
  5. o(n^2) o(n^3)……o(n^k)
  6. o(2^n) o(3^n)……0(k^n)
  7. o(n!)

###2.额外空间复杂度

在实现一个算法的过程中，除了输入参数与输出结果所需要必要的开辟出来的空间之外，你的算法流程还必须开辟额外的空间才能让你的流程执行下去，这部分空间就是额外空间。

如果你的流程只需要开辟几个有限的变量，额外空间复杂度就是o(1)。

###3.常数项时间

####常见的常数时间操作：
1.常见的算数运算符（+，-，*，/，%等）
2.常见的位运算（<<，>>，&，|，等）
3.赋值，比较，自增，自减操作等
4.数组的寻址操作（因为数组存储数据的方式是连续的）

####算法流程的总操作数量与样本数量之间的表达式关系的确定：
1.想象数据算法流程所处理的数据状况，按照最差的来。
2.把整个流程彻底拆分成一个个基本动作，保证每一个动作都是常熟时间的操作。
3.如果数据量为N，则观察基本动作的数量与N的关系

##总结

1.在衡量算法的标准中，时间复杂度最为重要，额外空间复杂度次之，而常数项时间最无关紧要。

2.判断两个算法的好坏，第一标准就是时间复杂度，时间复杂度大的算法一定不如时间复杂度小的算法；第二标准是额外空间复杂度，在时间复杂度相等的情况下，哪个算法需要额外申请的空间越小，它一定越好；如果两个算法的时间复杂度和额外空间复杂度都相同，我们只需要提供大量数据让程序跑起来，看哪个算法的用时最少。

3.我们不进行常数项分析的原因是这不仅需要大量的分析而且分析的结果也是不一定正确的。因为常数操作里也是分快慢的，比如位运算就一定要比加减乘除等其他运算快。所以我们不如直接提供大量数据让程序跑起来，让事实说话。

1.你想测试的方法a
2.实现复杂度不好但是容易实现的方法b
3.实现一个随机样本产生器
